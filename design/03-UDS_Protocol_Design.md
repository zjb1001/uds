## 1. UDS 协议基础

### 1.1 协议概述

UDS (Unified Diagnostic Services) 定义在 ISO 14229-1 标准中，是现代汽车诊断的统一标准。本系统实现 ISO 14229-1 核心诊断服务的完整子集，包括会话管理、数据读写、安全访问、刷写、诊断程序执行及通信控制等。

#### 支持的服务清单 (17 项核心服务)

| 服务ID | 服务名称 | 英文名 | 实现优先级 |
|--------|--------|--------|----------|
| 0x10 | 诊断会话控制 | Diagnostic Session Control | P0 (必需) |
| 0x11 | ECU 重启 | ECU Reset | P0 (必需) |
| 0x14 | 清除诊断信息 | Clear Diagnostic Information | P0 (必需) |
| 0x19 | 读取故障代码 | Read DTC Information | P0 (必需) |
| 0x22 | 按标识符读取数据 | Read Data by Identifier | P0 (必需) |
| 0x23 | 按地址读取数据 | Read Memory by Address | P0 (必需) |
| 0x24 | 读取缩放数据 | Read Scaling Data Information | P1 (优先) |
| 0x27 | 安全访问/解锁 | Security Access | P0 (必需) |
| 0x2E | 按标识符写入数据 | Write Data by Identifier | P1 (优先) |
| 0x28 | 通信控制 | Communication Control | P1 (优先) |
| 0x31 | 诊断程序执行 | Routine Control | P1 (优先) |
| 0x34 | 请求下载 | Request Download | P0 (必需) |
| 0x35 | 请求上传/读取 | Request Upload | P0 (必需) |
| 0x36 | 数据传输 | Transfer Data | P0 (必需) |
| 0x37 | 传输退出 | Request Transfer Exit | P0 (必需) |
| 0x3D | 按地址写入内存 | Write Memory by Address | P1 (优先) |
| 0x3E | 测试器保活 | Tester Present | P0 (必需) |
| 0x85 | DTC 设置控制 | Control DTC Setting | P1 (优先) |

### 1.2 诊断寻址 (Diagnostic Addressing)

#### 物理寻址 (Physical Addressing)
物理寻址用于点对点通信，诊断仪发送请求到特定 ECU，该 ECU 返回响应。

**CAN ID 映射规则**:
```
请求方向 (诊断仪 → ECU):
  ECU请求CAN_ID = 0x600 + ECU_ID
  示例: ECU 0 请求 ID = 0x600, ECU 1 请求 ID = 0x601

响应方向 (ECU → 诊断仪):
  ECU响应CAN_ID = 0x680 + ECU_ID  
  示例: ECU 0 响应 ID = 0x680, ECU 1 响应 ID = 0x681
```

**特点**:
- 一个诊断会话仅一个 ECU 响应
- 用于编程、刷写、单 ECU 诊断
- 需要明确的 ECU ID 配置

#### 功能寻址 (Functional Addressing)
功能寻址用于广播通信，多个 ECU 可接收但仅配置的目标 ECU 响应。

**CAN ID 定义**:
```
功能请求 (广播):
  功能CAN_ID = 0x7DF (保留标准值)
  所有 ECU 监听此 ID
  
分类功能寻址:
  按模块/系统的功能地址: 0x7E0 - 0x7EF (16 个功能组)
```

**响应规则**:
- ECU 评估是否为自己的功能地址
- 默认仅目标 ECU 响应，防止"总线风暴"
- 可通过配置禁用功能寻址响应

### 1.3 ISO-TP 传输层 (ISO 15765-2)

#### ISO-TP 帧类型与格式

ISO-TP 将 UDS 长报文分解为多个 CAN 帧，通过以下帧类型完成分段与重组：

**单帧 (Single Frame, SF)**
```
CAN Data: 0x0N DD DD DD DD DD DD DD
          └─┘ └────────────────────┘
          PCIID  服务数据 (最多 7 字节)

N = 数据长度 (0-7)
最大数据: 7 字节 (不需分段)
```

**首帧 (First Frame, FF)**
```
CAN Data: 0x1L LL DD DD DD DD DD DD
          └─┘ └┘ └────────────────────┘
          PCIID  长度    前 6 字节数据

LL = 总长度高字节 (最多 4095 字节)
首帧提供 6 字节数据，剩余数据通过连续帧传送
```

**连续帧 (Consecutive Frame, CF)**
```
CAN Data: 0x2N DD DD DD DD DD DD DD
          └─┘ └────────────────────┘
          PCIID  后续数据 (7 字节)

N = 序列号 (0-15，循环滚动)
连续帧每帧最多 7 字节
```

**流控帧 (Flow Control, FC)**
```
CAN Data: 0x3F FS BS STmin XX XX XX
          └─┘ └─ ──┘ └───┘ 
          PCIID  流状 块大  等待时间

FS = Flow Status:
  0x00: Continue To Send (CTS, 继续发送)
  0x01: Wait (WT, 等待，发送方应重试)
  0x02: Overflow (OVF, 接收缓冲溢出，中止传输)

BS (BlockSize) = 发送 BS 帧后，发送方等待下一个 FC 帧
  0x00 = 无限制，一直发送直到完成

STmin (Separation Time minimum) = 连续帧最小间隔 (毫秒)
  范围: 0-127ms (虚拟 CAN 可忽略，物理 CAN 需遵守)
```

#### ISO-TP 传输流程示例

**发送多帧数据** (例: 100 字节数据):
```
发送方 (诊断仪)              接收方 (ECU)
    │                          │
    ├─ 发送 FF (6B) ────────>  │
    │                          ├─ 组装 FF 信息
    │                          ├─ 计算剩余帧数 (14 帧)
    │                          ├─ 按 BS/STmin 准备
    │  <────── FC (CTS) ────── ├─ 发送 FC
    │
    ├─ 发送 CF[1] (7B) ───-->  │
    │  (STmin = 5ms)           ├─ 缓存, 计算校验
    │  ... (重复 13 次)
    │
    ├─ 发送 CF[14] (2B) ───-->  │
    │                          ├─ 重组完成，校验通过
    │                          ├─ 处理 UDS 服务
    │  <────── SF/FF ────────── ├─ 返回响应
```

#### 超时与重试参数

| 参数 | 符号 | 默认值 | 说明 |
|------|------|--------|------|
| 单帧响应超时 | P2 | 50ms | 常规会话，等待响应 |
| 长时操作超时 | P2* | 2000ms | 刷写/例程，可返回 NRC 0x78 |
| 会话空闲超时 | S3 | 10000ms | 无 Tester Present 则回 Default |
| 多帧段间隔 | STmin | 5ms | 连续帧发送间隔 |
| 流控等待超时 | N_Bs | 1000ms | 等待 FC 帧超时 |
| 连续帧接收超时 | N_Cr | 1000ms | 等待 CF 帧超时 |

**配置说明**: 所有超时值需与 [01-System_Requirements_Specification.md](01-System_Requirements_Specification.md) FR-TRANS-003 一致，并在运行时可配置。

### 1.4 UDS 报文基本格式

#### 肯定响应 (Positive Response)

```
SID + 0x40 = 响应 SID

示例: 
  请求:  0x22 F1 90 (读取 VIN)
  响应:  0x62 F1 90 [17字节 VIN数据]
```

#### 否定响应 (Negative Response)

```
0x7F | Original_SID | NRC_Code

示例: 
  请求:  0x22 F1 90 (权限不足)
  响应:  0x7F 22 33 (NRC=0x33 securityAccessDenied)
```

#### 抑制肯定响应 (Suppress Positive Response)

部分服务支持通过设置 SubFunction Bit7 来抑制肯定响应：
```
SubFunction 格式:
  Bit 7 = 1: 抑制肯定响应 (ECU 不返回肯定响应，但仍执行操作)
  Bit 7 = 0: 正常响应 (ECU 返回肯定响应)
  Bit 0-6: 功能选择位

支持的服务:
  - 0x3E Tester Present: suppressPosRspMsgIndicationBit
  - 0x31 Routine Control: routineOptionRecord
  
约束:
  - 即使抑制肯定响应，ECU 必须执行操作并维持状态
  - 负响应 (错误) 始终返回，不受抑制
  - 超时窗口仍需遵守 (无响应 = 超时)
```

## 2. 诊断会话与基础控制服务

### 2.0 服务 0x11: ECU Reset (ECU 重启)

#### 功能概述
请求 ECU 执行重启操作，返回初始状态。重启后会话自动回到 Default Session。

#### 报文格式

**请求格式**:
```
┌─────────────┬──────────────┐
│ SID (0x11)  │ resetType    │
│ 1 Byte      │ 1 Byte       │
└─────────────┴──────────────┘
```

**重启类型定义**:
| 重启类型 | 值 | 说明 | 应用场景 |
|---------|----|----|--------|
| Hard Reset | 0x01 | 硬复位（电源循环模拟） | 刷写完成后强制重启 |
| Key Off/On Reset | 0x02 | 点火循环重启 | 标准诊断重启 |
| Soft Reset | 0x03 | 软重启（不断电） | 恢复正常工作模式 |
| Enable Rapid Power Shut Down | 0x04 | 启用快速关闭 | 电源管理（可选） |
| Disable Rapid Power Shut Down | 0x05 | 禁用快速关闭 | 电源管理（可选） |

**响应格式** (重启前发送):
```
┌─────────────┬──────────────┐
│ SID (0x51)  │ resetType    │
│ 1 Byte      │ 1 Byte       │
└─────────────┴──────────────┘

说明: ECU 必须在重启前返回肯定响应，确认命令已接收
```

#### 处理流程

```
请求 0x11 resetType
    ↓
┌─────────────────────────────┐
│ 验证重启权限                │
│ - 检查当前会话              │
│ - 检查重启类型合法性        │
└─────────────────────────────┘
    ↓ (验证失败)
返回 0x7F 11 NRC_CODE
    │
    ↓ (验证成功)
┌─────────────────────────────┐
│ 立即返回肯定响应            │
│ 0x51 + resetType            │
└─────────────────────────────┘
    ↓
┌─────────────────────────────┐
│ 清理资源                    │
│ - 关闭所有活动会话          │
│ - 清理传输上下文            │
│ - 清理刷写状态机            │
│ - 保存关键数据              │
└─────────────────────────────┘
    ↓
┌─────────────────────────────┐
│ 执行重启操作                │
│ - 根据 resetType 选择方式   │
│ - (在虚拟环境中模拟延迟)    │
│ - 重新初始化模块            │
└─────────────────────────────┘
    ↓
┌─────────────────────────────┐
│ 重启完成                    │
│ - 返回 Default Session      │
│ - 所有状态复位              │
│ - 安全级别清零              │
└─────────────────────────────┘
```

#### 约束条件

| 项目 | 值 | 说明 |
|------|-----|------|
| 权限要求 | 所有会话 | 可从任何会话发起 |
| 执行时间 | ≤ 100ms | 必须快速返回肯定响应 |
| 重启延迟 | 100-500ms | 虚拟环境可配置 |
| 响应 | 必须在重启前 | 诊断仪需等待响应 |
| 会话影响 | 回到 Default | 自动重置所有会话 |
| 状态清除 | 完全重置 | 除持久化数据外清零 |

#### NRC 错误处理

| 触发条件 | NRC 码 | 说明 |
|---------|--------|------|
| 非法重启类型 | 0x12 | subFunctionNotSupported |
| 报文长度错误 | 0x13 | incorrectMessageLengthOrInvalidFormat |
| 重启被禁用 | 0x22 | conditionsNotCorrect |
| 刷写进行中 | 0x24 | requestSequenceError (禁止中断) |

### 2.1 服务 0x10: Diagnostic Session Control

#### 功能概述
切换诊断会话类型，控制 ECU 工作模式

#### 报文格式

**请求格式**:
```
┌─────────────┬──────────────┐
│ SID (0x10)  │ sessionType  │
│ 1 Byte      │ 1 Byte       │
└─────────────┴──────────────┘
```

**会话类型定义**:
| 会话类型 | 值 | 说明 |
|---------|----|----|
| defaultSession | 0x01 | 标准工作模式 |
| programmingSession | 0x10 | 编程模式，需要解锁 |
| extendedDiagnosticSession | 0x03 | 扩展诊断 |
| safetySystemDiagnosticSession | 0x04 | 安全系统诊断 |

**响应格式**:
```
┌─────────────┬──────────────┬──────────────┬──────────┐
│ SID (0x50)  │ sessionType  │ P2_Server    │ P2*_Server│
│ 1 Byte      │ 1 Byte       │ 2 Bytes      │ 2 Bytes  │
└─────────────┴──────────────┴──────────────┴──────────┘

P2_Server: 默认超时时间 (毫秒)，范围 10-1000ms
P2*_Server: 加长超时时间，用于长时间操作
```

#### 处理流程

```
请求 0x10 sessionType
    ↓
┌─────────────────────────────┐
│ 验证权限                    │
│ - 检查当前会话允许的操作    │
│ - 检查安全级别              │
└─────────────────────────────┘
    ↓
┌─────────────────────────────┐
│ 验证会话转换合法性          │
│ - Default → Programming 需要│
│ - 安全解锁                  │
└─────────────────────────────┘
    ↓
┌─────────────────────────────┐
│ 更新会话状态                │
│ - 保存新会话类型            │
│ - 重置超时计时器            │
│ - 更新权限表                │
└─────────────────────────────┘
    ↓
返回 0x50 + sessionType + 超时参数
```

#### 会话权限矩阵

| 操作/服务 | Default | Extended | Programming | Safety |
|----------|---------|----------|-------------|--------|
| 读取数据 (0x22) | ✓ | ✓ | ✓ | ✓ |
| 写入数据 (0x2E) | ✗ | ✗ | ✓ | ✗ |
| 清除 DTC (0x14) | ✗ | ✗ | ✓ | ✗ |
| 刷写 (0x34-0x37) | ✗ | ✗ | ✓ | ✗ |
| 安全解锁 (0x27) | ✗ | ✓ | ✓ | ✓ |
| 诊断程序 (0x31) | ✗ | ✓ | ✓ | ✓ |

#### 超时管理

```
Programming Session:
  ├─ 无操作超时: 30 分钟
  ├─ 刷写操作超时: 60 秒
  └─ 单帧数据超时: 5 秒

Extended Diagnostic Session:
  ├─ 无操作超时: 10 分钟
  └─ 操作超时: 10 秒

Safety System Session:
  └─ 无操作超时: 5 分钟
```

## 2.2 服务 0x27: Security Access (安全访问/解锁)

#### 功能概述
在进入编程会话前需要安全验证。通过种子-密钥机制实现安全认证，防止未授权的诊断操作。

#### 安全级别定义

| 级别 | 说明 | 应用场景 | 权限 |
|------|------|---------|------|
| 1 | 读取与诊断 | 查看故障码、温度等 | 不需要解锁 |
| 2 | 标定修改 | 修改参数、标定 | 需 Extended Diagnostic Session |
| 3 | 编程刷写 | ECU 固件刷写、内存写入 | 需 Programming Session + 解锁 |

#### 报文格式

**请求种子 (subFunction 0x01/0x03/0x05)**:
```
┌─────────────┬────────────┬──────────────┐
│ SID (0x27)  │ SubFunc    │ SecurityLevel│
│ 1 Byte      │ 1 Byte     │ 1 Byte       │
└─────────────┴────────────┴──────────────┘

SubFunction 说明:
  0x01 = requestSeed for Level 1
  0x03 = requestSeed for Level 2  
  0x05 = requestSeed for Level 3 (编程解锁)
```

**种子响应 (0x61)**:
```
┌─────────────┬────────────┬──────────────┬─────────────┐
│ SID (0x61)  │ SubFunc+1  │ SeedLength   │ Seed Data   │
│ 1 Byte      │ 1 Byte     │ 1-2 Bytes    │ 可变        │
└─────────────┴────────────┴──────────────┴─────────────┘

Seed Data 长度: 4-16 字节 (随机或伪随机)
Seed 有效期: 10 秒，超期需重新请求
```

**发送密钥 (subFunction 0x02/0x04/0x06)**:
```
┌─────────────┬────────────┬──────────────┬──────────────┐
│ SID (0x27)  │ SubFunc    │ KeyLength    │ Key Data     │
│ 1 Byte      │ 1 Byte     │ 1-2 Bytes    │ 可变         │
└─────────────┴────────────┴──────────────┴──────────────┘

SubFunction 说明:
  0x02 = sendKey for Level 1
  0x04 = sendKey for Level 2
  0x06 = sendKey for Level 3

Key Data 长度: 应与 Seed 对应计算长度一致
```

**成功响应 (0x67)**:
```
┌─────────────┬────────────┐
│ SID (0x67)  │ SubFunc+1  │
│ 1 Byte      │ 1 Byte     │
└─────────────┴────────────┘

说明: 解锁成功，该安全级别权限已获得
```

#### 密钥生成算法

系统支持分级密钥生成算法：

**初期 (简化模式)**:
```
Key = Seed XOR 0xABCD1234
  
示例:
  Seed = 0x12345678
  Key  = 0x12345678 XOR 0xABCD1234 = 0xB9F84A4C
```

**扩展 (HMAC-SHA256)**:
```
Key = HMAC-SHA256(Seed, MasterSecret)[0:8]
  
其中 MasterSecret 为系统预配密钥 (可在配置文件中设置)

优势: 安全性强，防止种子公开导致密钥泄露
```

**可配置算法**:
- XOR (快速，用于测试)
- HMAC-SHA1 / SHA256 (标准安全)
- 自定义算法插件（通过配置文件指定)

#### 处理流程

```
请求 0x27 0x01 Level
    ↓
┌─────────────────────────────┐
│ 检查会话与安全状态          │
│ - 检查当前会话允许级别      │
│ - 检查是否已解锁            │
└─────────────────────────────┘
    ↓ (已解锁)
返回 NRC 0x24 requestSequenceError
    │
    ↓ (未解锁，继续)
┌─────────────────────────────┐
│ 生成随机种子                │
│ - 调用种子生成器            │
│ - 记录生成时间与有效期      │
│ - 设置重试计数器            │
└─────────────────────────────┘
    ↓
返回 0x61 + Seed (长度 4-16B)
    │
    ├─────────────────────────> (诊断仪计算密钥)
    │
发送 0x27 0x02 Key
    ↓
┌─────────────────────────────┐
│ 验证种子有效性              │
│ - 检查生成时间 (< 10秒)     │
│ - 计数器是否重置            │
└─────────────────────────────┘
    ↓ (种子过期)
返回 NRC 0x37 requiredTimeDelayNotExpired
    │
    ↓ (种子有效)
┌─────────────────────────────┐
│ 计算预期密钥                │
│ - 使用相同算法与种子        │
│ - 生成期望的密钥值          │
└─────────────────────────────┘
    ↓
┌─────────────────────────────┐
│ 密钥比对                    │
│ - 常数时间比较 (防时序攻击) │
│ - 记录比对结果              │
└─────────────────────────────┘
    ↓ (密钥错误)
┌─────────────────────────────┐
│ 失败处理                    │
│ - 重试计数器 +1             │
│ - 检查是否超过阈值 (3 次)   │
└─────────────────────────────┘
    ├─ 未超阈值 → 返回 NRC 0x35 invalidKey
    │
    └─ 超过阈值 → 触发锁定
       ├─ 设置锁定计时器 (300 秒)
       ├─ 禁止后续密钥尝试
       └─ 返回 NRC 0x36 exceededNumberOfAttempts
    │
    ↓ (密钥正确)
┌─────────────────────────────┐
│ 更新安全状态                │
│ - 记录解锁级别              │
│ - 清除重试计数器            │
│ - 标记为已授权              │
└─────────────────────────────┘
    ↓
返回 0x67 + SubFunc+1
```

#### 约束与保护措施

**防暴力攻击**:
```
最大尝试次数: 3 次
失败后锁定时间: 300 秒 (5 分钟)
锁定期间操作: 返回 NRC 0x36 exceededNumberOfAttempts

示例:
  1st try: 密钥错误 → NRC 0x35 invalidKey (重试计数=1)
  2nd try: 密钥错误 → NRC 0x35 invalidKey (重试计数=2)
  3rd try: 密钥错误 → NRC 0x35 invalidKey (重试计数=3, 触发锁定)
  4th try: 被锁定 → NRC 0x36 exceededNumberOfAttempts (锁定计时=300s)
  ...等待 300 秒...
  重试可用: 重试计数器与锁定解除
```

**种子有效期**:
- 生成后 10 秒内必须发送密钥
- 超期需重新请求种子

**NRC 错误码**:

| NRC | 触发条件 | 说明 |
|-----|---------|------|
| 0x12 | 不支持的子功能 | SubFunction 非 0x01/0x03/0x05/0x02/0x04/0x06 |
| 0x13 | 格式错误 | 报文长度或格式不符 |
| 0x22 | 会话条件不允许 | 当前会话不允许请求的级别 |
| 0x24 | 请求序列错误 | 已解锁，重复解锁 |
| 0x35 | 无效密钥 | 密钥验证失败，计数+1 |
| 0x36 | 超过尝试次数 | 3 次失败，触发 300s 锁定 |
| 0x37 | 时间延迟未到期 | 种子过期或锁定期未满 |

## 3. 诊断故障码服务

### 3.1 服务 0x19: Read DTC Information

#### 功能概述
读取 ECU 的诊断故障码及其状态信息

#### DTC 格式定义

**DTC 编码** (3 字节):
```
字节 0 (MSB): ┌─────────────┐
            │ H H       │  H = High Byte
              ├─────────────┤  包含 DTC 类别信息
字节 1: │ M M M M │  M = Middle Byte
        └─────────────┘
字节 2 (LSB): │ L L L L │  L = Low Byte

例子: P0100
P = 动力总成 (Powertrain) - 0xP
01 = 燃油和进气系统 - 0x01
00 = MAF 传感器电路故障 - 0x00
```

**DTC 状态** (1 字节):
```
Bit 7-6: Reserved
Bit 5:   CDTC (Clear DTC enabled)
Bit 4:   TFTOC (Tester present or one cycle)
Bit 3:   PDTC (Pending DTC)
Bit 2:   CDTC (Confirmed DTC)
Bit 1:   DTC Warn Indicator On
Bit 0:   Reserved
```

#### 子功能列表

| SubFunc | 说明 | 返回数据 |
|---------|------|--------|
| 0x01 | Report number of DTC by Status Mask | DTC 计数 (2 字节) |
| 0x02 | Report DTC and Status Record | DTC (3B) + Status (1B) |
| 0x03 | Report DTC Snapshot Record | DTC + 快照数据 |
| 0x04 | Report DTC Extended Data Record | DTC + 扩展数据 |
| 0x14 | Report number of DTC | DTC 总数 |
| 0x15 | Report Extended Data Record by DTC | 指定 DTC 的扩展数据 |
| 0x42 | Report DTC Enabled Status | DTC 使能状态 |

#### 报文格式

**请求 0x19-0x01** (按状态掩码查询 DTC 数量):
```
┌─────────────┬─────────────┬─────────────┐
│ SID (0x19)  │ SubFunc     │ StatusMask  │
│ 1 Byte      │ 1 Byte      │ 1 Byte      │
│             │ 0x01        │ 0xFF=全部   │
└─────────────┴─────────────┴─────────────┘
```

**响应**:
```
┌─────────────┬─────────────┬──────────────┬──────────────┐
│ SID (0x59)  │ SubFunc     │ StatusAvail  │ DTCAndMask   │
│ 1 Byte      │ 1 Byte      │ 1 Byte       │ 2 Bytes      │
│             │ 0x01        │ (可选)       │ (可选)       │
└─────────────┴─────────────┴──────────────┴──────────────┘
```

**请求 0x19-0x02** (获取 DTC 和状态):
```
┌─────────────┬─────────────┬─────────────┐
│ SID (0x19)  │ SubFunc     │ StatusMask  │
│ 1 Byte      │ 1 Byte      │ 1 Byte      │
│             │ 0x02        │             │
└─────────────┴─────────────┴─────────────┘
```

**响应**:
```
┌─────────────┬──────────────────────────────────┐
│ SID (0x59)  │ {DTC (3B) + Status (1B)} × N     │
│ 1 Byte      │ 可能多条 DTC 记录                │
└─────────────┴──────────────────────────────────┘
```

#### DTC 存储和管理

**最大 DTC 数**: 100 个
**存储位置**: 虚拟非易失性存储器
**初始化**: 系统启动时加载配置的 DTC 列表

## 5. 数据读写与寻址服务

### 5.0 服务 0x23: Read Memory by Address (按地址读取内存)

#### 功能概述
按指定的起始地址与长度读取 ECU 内存数据。与 0x22 (按标识符读取) 互补，提供更灵活的内存访问方式。

#### 报文格式

**请求格式**:
```
┌─────────────┬──────────────┬──────────┬────────────┐
│ SID (0x23)  │ AddrLen      │ Address  │ Length     │
│ 1 Byte      │ 1 Byte       │ 可变     │ 可变       │
└─────────────┴──────────────┴──────────┴────────────┘

AddrLen: 地址和长度格式标识
  Bit 7-4: 长度字段长度 (字节数)
  Bit 3-0: 地址字段长度 (字节数)
  
示例: 
  0x24 = 地址 2 字节, 长度 4 字节
  0x23 = 地址 2 字节, 长度 3 字节
  0x44 = 地址 4 字节, 长度 4 字节

Address: 起始地址 (大端序)
  示例: 0x08000000 (应用区起始地址)

Length: 读取长度 (字节数)
  范围: 1-4095 字节 (单帧/多帧)
```

**响应格式** (肯定响应):
```
┌─────────────┬──────────────────┐
│ SID (0x63)  │ MemoryData       │
│ 1 Byte      │ 可变 (1-4095B)   │
└─────────────┴──────────────────┘
```

#### 权限与保护规则

| 会话 | Default | Extended | Programming |
|------|---------|----------|-------------|
| 公开区 (0xF1xx) | ✓ | ✓ | ✓ |
| 应用区 (0x08000-0x3FFFF) | ✗ | ✗ | ✓ |
| 参数区 (0x40000-0x47FFF) | ✗ | ✗ | ✓ |
| Bootloader 区 (受保护) | ✗ | ✗ | 仅读 |

#### NRC 错误码

| NRC | 触发条件 |
|-----|---------|
| 0x13 | 格式错误，AddrLen 或地址/长度格式不符 |
| 0x22 | 权限不足，会话不允许访问该地址 |
| 0x31 | 地址或长度超出范围 |
| 0x70 | uploadDownloadNotAccepted (缓冲不足) |

### 5.1 服务 0x24: Read Scaling Data Information (读取缩放数据)

#### 功能概述
读取 DID (数据标识符) 对应的缩放信息 (单位、精度、范围等)。通常与 0x22 配合使用，用于诊断数据的解释和转换。

#### 应用场景

```
0x22 读取 DID:
  ├─ DID 0xF910 → 原始值: 0x2710 (十进制 10000)
  │
0x24 读取缩放信息:
  ├─ 缩放系数: 0.01
  ├─ 偏移: 0
  ├─ 单位: "°C"
  │
实际值 = (原始值 * 0.01) = 100°C
```

#### 报文格式

**请求格式**:
```
┌─────────────┬────────────┐
│ SID (0x24)  │ DID        │
│ 1 Byte      │ 2 Bytes    │
└─────────────┴────────────┘

说明: 与 0x22 请求格式一致，但不支持多个 DID
```

**响应格式**:
```
┌─────────────┬──────────────┬─────────────┬──────────┬──────────┐
│ SID (0x64)  │ DID          │ Scale/Prec  │ Unit     │ Min/Max  │
│ 1 Byte      │ 2 Bytes      │ 2 Bytes     │ 可变     │ 可变     │
└─────────────┴──────────────┴─────────────┴──────────┴──────────┘

Scale/Precision: 缩放系数 (定点数格式)
  示例: 0x0064 = 100 (表示 * 0.01)

Unit: 单位字符串 (可选，长度可变)
  示例: "°C", "mV", "RPM"

Min/Max: 范围限制 (可选)
  Min: 最小值, Max: 最大值
```

#### 支持的 DID 与缩放表

| DID | 名称 | 缩放系数 | 单位 | 范围 |
|-----|------|---------|------|------|
| 0xF910 | ECU Temperature | 0.01 | °C | -40 ~ 125 |
| 0xF911 | Supply Voltage | 0.001 | V | 5 ~ 20 |
| 0xF912 | System State | 1.0 | - | 0 ~ 255 |
| 0xF401 | Compile Date | 1.0 | - | - |

#### 权限与约束

- 权限: 默认会话可读所有 DID 缩放信息
- 不存在的 DID: 返回 NRC 0x31 requestOutOfRange
- 无缩放信息的 DID: 返回默认缩放 (系数=1, 偏移=0)

```
┌─────────────┬────────────┬───────────┬─────┬────────────┬──────────┐

└─────────────┴────────────┴───────────┴─────┴────────────┴──────────┘
```

**响应格式** (否定响应示例):
```
0x7F 22 13 - 格式不正确
0x7F 22 31 - DID 超出范围
0x7F 22 33 - 权限不足
```

#### 处理流程

```
请求 0x22 DID_1 [DID_2 ...]
    ↓
┌─────────────────────────────┐
│ 参数验证                    │
│ - 检查请求长度              │
│ - 检查 DID 格式              │
└─────────────────────────────┘
    ↓ (验证失败)
返回 0x7F 22 13 (格式错误)
    │
    ↓ (验证成功)
┌─────────────────────────────┐
│ 权限检查                    │
│ - 检查当前会话              │
│ - 检查安全级别              │
│ - 检查 DID 访问权限          │
└─────────────────────────────┘
    ↓ (权限检查失败)
返回 0x7F 22 33 (权限不足)
    │
    ↓ (权限检查成功)
┌─────────────────────────────┐
│ 数据检索                    │
│ - 对每个 DID:               │
│   └─ 查询数据存储           │
│   └─ 执行数据转换 (若需要)  │
│   └─ 格式化数据             │
└─────────────────────────────┘
    ↓
┌─────────────────────────────┐
│ 响应组装                    │
│ - 组装 DID + Data           │
│ - 计算响应长度              │
│ - 检查响应是否超过单帧      │
└─────────────────────────────┘
    ↓
返回 0x62 + DID_1 + Data_1 + ...
```

#### 多帧响应处理

如果响应数据超过单帧容量（7 字节报文数据 - SID 1B = 6B）：

```
响应数据总长 > 6 Bytes
    ↓
分割成多帧发送:
  First Frame (FF): 0x10 + Length (2B) + 前 5 字节数据
  Consecutive Frame (CF): 0x2n + 后续数据

示例:
请求: 0x22 F1 90 (VIN, 17 字节)
响应:
  FF: 0x10 12 62 F1 90 [5 字节 VIN]
  CF: 0x21 [12 字节 VIN 剩余部分]
```

```
## 8. 超时与 NRC 约定

### 8.1 超时参数 (建议值)

| 场景 | 符号 | 建议值 | 说明 |
|------|------|------|------|
| 单帧响应超时 | P2 | 50-100 ms | 默认会话/扩展会话 |
```

## 8. 错误处理与 NRC 码详表

### 8.1 NRC 完整码表 (根据 ISO 14229-1)

| NRC | 十进制 | 名称 | 触发场景 | 推荐处理 |
|-----|--------|------|--------|---------|
| 0x10 | 16 | generalReject | 通用拒绝，具体原因不确定 | 记录日志，重试或回报给用户 |
| 0x11 | 17 | serviceNotSupported | 服务 ID (SID) 未实现或不支持 | 检查 ECU 功能，升级诊断软件 |
| 0x12 | 18 | subFunctionNotSupported | 子功能 (SubFunc) 无效 | 验证请求参数 |
| 0x13 | 19 | incorrectMessageLengthOrInvalidFormat | 报文长度或格式错误 | 检查报文结构，重新发送 |
| 0x14 | 20 | responseTooLong | 响应数据超过传输容量 | 分块读取，或等待 P2* 超时 |
| 0x21 | 33 | busyRepeatRequest | ECU 忙碌，稍后重试 | 等待 P2* 时间后重试 |
| 0x22 | 34 | conditionsNotCorrect | 操作条件不满足（会话、权限不足） | 切换到正确的会话，或进行安全解锁 |
| 0x24 | 36 | requestSequenceError | 请求顺序错误（如未先发 0x34 就发 0x36） | 按正确流程重新发送请求 |
| 0x25 | 37 | noMemoryAvailable | ECU 内存不足 | 清除不必要的数据，重试 |
| 0x26 | 38 | negativeDynamicDataIdentifier | 动态 DID 配置失败 | 检查 DID 定义，重新配置 |
| 0x31 | 49 | requestOutOfRange | 请求参数超出范围（地址、DID、长度） | 验证参数范围，检查 ECU 文档 |
| 0x33 | 51 | securityAccessDenied | 安全访问被拒绝（未解锁或权限不足） | 执行安全解锁 (0x27)，重试 |
| 0x35 | 53 | invalidKey | 密钥验证失败 | 重新请求种子，计算正确密钥，重试 |
| 0x36 | 54 | exceededNumberOfAttempts | 超过最大尝试次数，安全锁定 | 等待锁定解除 (通常 300s)，再重试 |
| 0x37 | 55 | requiredTimeDelayNotExpired | 所需时间延迟未到期 | 等待足够时间（如种子有效期），再重试 |
| 0x50 | 80 | enableRxAndTx | 恢复接收与发送 (通信控制恢复) | - |
| 0x51 | 81 | enableRxAndDisableTx | 启用接收，禁用发送 | - |
| 0x52 | 82 | disableRxAndEnableTx | 禁用接收，启用发送 | - |
| 0x53 | 83 | disableRxAndTx | 禁用接收与发送 | - |
| 0x70 | 112 | uploadDownloadNotAccepted | 上传/下载操作不被接受 | 检查会话、权限，重试 |
| 0x71 | 113 | transferDataSuspended | 数据传输被暂停 | 恢复传输或重新开始 |
| 0x72 | 114 | generalProgrammingFailure | 编程失败（刷写、擦除等） | 检查刷写参数，重试或寻求技术支持 |
| 0x73 | 115 | wrongBlockSequenceCounter | 连续帧序列号错误 | 重新发送或刷新连接 |
| 0x78 | 120 | requestCorrectlyReceivedResponsePending | 请求已接收，响应待定（需等待） | 等待 P2* 时间，接收最终响应 |
| 0x7E | 126 | subFunctionNotSupportedInActiveSession | 该子功能在当前会话不支持 | 切换到支持该子功能的会话 |
| 0x7F | 127 | serviceNotSupportedInActiveSession | 该服务在当前会话不支持 | 切换会话或检查权限 |

### 8.2 NRC 处理决策表

| 错误类型 | 推荐策略 | 示例 NRC | 诊断仪动作 |
|---------|--------|--------|----------|
| 参数错误 | 修正后重试 | 0x13, 0x31 | 验证请求参数，修正后重新发送 |
| 权限/会话 | 切换会话或解锁 | 0x22, 0x33, 0x7F | 进入正确会话或进行安全解锁 |
| 密钥失败 | 锁定处理 | 0x35, 0x36 | 等待锁定解除，重新尝试 |
| 时序问题 | 等待重试 | 0x21, 0x37, 0x78 | 遵守 P2/P2* 超时，定时重试 |
| ECU 故障 | 故障诊断 | 0x72, 0x25 | 尝试复位或咨询技术支持 |
| 序列错误 | 重启流程 | 0x24 | 重新开始诊断流程 |

## 9. 服务总览与快速参考

### 9.1 服务分类总览

**管理与控制类** (4 项):
- 0x10 Diagnostic Session Control - 会话管理
- 0x11 ECU Reset - ECU 重启  
- 0x27 Security Access - 安全解锁
- 0x3E Tester Present - 测试器保活

**诊断查询类** (5 项):
- 0x14 Clear Diagnostic Information - 清除故障码
- 0x19 Read DTC Information - 读取故障码
- 0x22 Read Data by Identifier - 按标识符读数据
- 0x23 Read Memory by Address - 按地址读内存
- 0x24 Read Scaling Data Information - 读取缩放数据

**数据修改类** (3 项):
- 0x2E Write Data by Identifier - 按标识符写数据
- 0x3D Write Memory by Address - 按地址写内存
- 0x31 Routine Control - 诊断程序执行

**刷写与传输类** (4 项):
- 0x34 Request Download - 请求下载(刷写前置)
- 0x35 Request Upload - 请求上传(读取前置)
- 0x36 Transfer Data - 数据传输
- 0x37 Request Transfer Exit - 传输完成

**通信控制类** (2 项):
- 0x28 Communication Control - 通信控制
- 0x85 Control DTC Setting - DTC 设置控制
